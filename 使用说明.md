# 📖 My Wiki — 使用操作说明

> 本文档是知识库的日常使用手册，面向用户（你自己），说明如何与 AI Agent 协作维护这个知识库。

---

## 一、核心工作流程

```
[你发现好内容]
      ↓
  丢入 _inbox/
      ↓
  告诉 AI「编译 inbox」
      ↓
  AI 自动归档到 wiki/
      ↓
  你查询 / 提问 / 导出
```

---

## 二、日常操作步骤

### Step 1：收集素材

将以下任意类型的内容放入 `_inbox/` 目录：

| 素材类型 | 示例 | 建议格式 |
|---|---|---|
| 网页文章 | 博客、新闻 | `.md`（用剪藏工具转换）|
| PDF 文件 | 论文、报告 | `.pdf` |
| 个人笔记 | 想法、摘抄 | `.md` / `.txt` |
| 社交媒体帖子 | 推文串、知乎回答 | `.md` |
| 会议/语音记录 | 转录文字 | `.md` / `.txt` |

> **推荐工具**：[Obsidian Web Clipper](https://obsidian.md/clipper) 可一键将网页保存为 Markdown。

---

### Step 2：触发编译

打开 AI Agent（Claude Code / Gemini CLI / Cursor），提供以下上下文后发出指令：

**提供上下文的方式（二选一）：**

- **方式 A**：将 `.llm-wiki.md` 内容作为系统 prompt 发给 AI
- **方式 B**：在对话开头说：「请阅读 `.llm-wiki.md` 并按其中的指令操作」

**然后发出指令：**

```
编译 inbox
```

> 编译的具体流程由 `.llm-wiki.md` 定义，无需记忆。AI 会自动完成读取、归类、合并、索引更新和归档。

---

### Step 3：查询与使用

知识库编译完成后，直接向 AI 提问即可：

```
猎头行业的 AI 机会有哪些？
```

```
整理一下我目前对 RAG 和 LLM Wiki 的对比认知
```

AI 会基于 `wiki/` 中已编译的知识回答，并标注引用来源。

## 三、完整指令速查表

### 🔧 维护类指令

| 指令 | 触发方式 | 输出位置 |
|---|---|---|
| 编译收件箱 | `编译 inbox` 或 `处理收件箱` | `wiki/` 各主题目录 |
| 健康检查 | `健康检查` 或 `lint` | `outputs/reports/health-check-YYYY-MM-DD.md` |
| 处理反馈 | `处理反馈` 或 `audit` | 直接修改对应 wiki 页面，归档至 `audit/resolved/` |
| 新建反馈 | `新建反馈 [页面路径] [问题描述]` | 生成规范 audit 文件到 `audit/` |
| 汇总开放问题 | `汇总开放问题` | `outputs/reports/open-questions-YYYY-MM-DD.md` |

### 📊 生成类指令

| 指令示例 | 说明 |
|---|---|
| `综合 猎头市场` | 跨页面整合该主题所有知识，生成综合报告 |
| `综合 AI 产品` | 同上，换主题 |
| `导出 幻灯片 猎头行业` | 生成 Marp 格式幻灯片到 `outputs/slides/` |
| `导出 报告 AI` | 生成详细报告到 `outputs/reports/` |
| `导出 摘要 产品` | 生成精简摘要到 `outputs/drafts/` |

### 🔍 聚焦类指令

| 指令示例 | 说明 |
|---|---|
| `聚焦 产品/猎头HUNTER` | 将检索范围锁定到该目录，避免跨领域知识干扰 |
| `退出聚焦` | 恢复全局检索模式 |

### ❓ 问答类指令

直接提问即可，无需特殊格式：

```
目前 wiki 里有哪些主题？
```
```
关于 XXX 我已经知道哪些，还缺什么？
```

## 四、目录结构说明

```
my_wiki/
├── README.md              ← 项目介绍（面向他人）
├── 使用说明.md             ← 本文件（面向自己）
├── .llm-wiki.md           ← AI 编译指令（核心配置，不要随意修改）
│
├── _inbox/                ← 📥 【你的操作区】丢入待处理素材
├── _raw/                  ← 📦 【只读归档】原始文件，不修改
│   ├── articles/
│   ├── papers/
│   ├── clippings/
│   └── threads/
│
├── wiki/                  ← 📖 【核心产出】AI 编译的结构化知识
│   ├── _index.md          ← 总目录索引（AI 自动维护）
│   ├── AI/
│   ├── 产品/
│   └── 学习与方法论/
│
├── outputs/               ← 📤 【最终产出】可对外使用的成品
│   ├── slides/
│   ├── reports/
│   └── drafts/
│
├── skills/                ← 🔧 【SKILL 插件】所有 AI Agent SKILL 统一存放于此
│   ├── anthropics-skills-docx/   ← Word 文档处理
│   ├── anthropics-skills-pptx/   ← PPT 演示文稿处理
│   └── debate-arguments/         ← 辩论论据构建
│
└── templates/             ← 📋 页面模板
```

## 五、推荐工作节奏

| 频率 | 操作 |
|---|---|
| **随时** | 发现好内容 → 丢入 `_inbox/` |
| **每周 1-2 次** | 执行 `编译 inbox`，处理积压素材 |
| **每月 1 次** | 执行 `健康检查`，维护知识质量 |
| **按需** | `综合` 或 `导出`，生成可用的报告/幻灯片 |

## 六、常见问题

**Q：`_inbox/` 里的文件处理后还在吗？**
A：AI 编译后会将原始文件移动到 `_raw/` 对应子目录，`_inbox/` 会被清空。

**Q：AI 会修改 `_raw/` 里的文件吗？**
A：不会。`_raw/` 是只读归档区，仅 `wiki/` 会被 AI 编写和更新。

**Q：wiki 页面信息过时了怎么办？**
A：将新的相关素材放入 `_inbox/`，重新编译时 AI 会自动合并更新。

**Q：如何新增一个主题目录？**
A：无需手动创建，编译 inbox 时若素材属于新主题，AI 会自动在 `wiki/` 下创建对应目录。

**Q：confidence 标签怎么理解？**
A：`high` = 有可靠来源；`medium` = 有部分依据；`low` = 推断性结论；`unverified` = 待核实。

**Q：Wiki 页面的标准格式是什么？**
A：详见 `.llm-wiki.md`「Wiki 页面格式规范」章节，或参考 `templates/` 目录下的模板文件。

## 七、SKILL 管理

> 所有 AI Agent 的 SKILL 插件统一安装在 `skills/` 目录下，方便集中管理和查找。

### 已安装 SKILL

| SKILL 名称 | 目录路径 | 功能 |
|:---|:---|:---|
| DOCX 文档处理 | `skills/anthropics-skills-docx/` | 创建、读取、编辑 Word (.docx) 文件，支持目录、标题、页码等专业格式 |
| PPTX 演示文稿 | `skills/anthropics-skills-pptx/` | 创建、读取、编辑 PowerPoint (.pptx) 文件，支持主题、布局、备注等 |
| HTML PPT Studio | `skills/html-ppt-skill/` | 生成专业静态 HTML 幻灯片，支持多种布局、动画、键盘导航，触发词：幻灯片/演讲稿/slides/deck |
| 辩论论据构建 | `skills/debate-arguments/` | 辩论赛论据三要素、五种论据类型、CEWI 论证框架、临场反驳四步法 |

### 安装新 SKILL 的规范

1. 将 SKILL 文件夹放入 `skills/` 目录
2. 确保 SKILL 目录内包含 `SKILL.md` 说明文件
3. 在上方表格中登记 SKILL 名称、路径和功能描述
4. **禁止**将 SKILL 安装到其他位置
